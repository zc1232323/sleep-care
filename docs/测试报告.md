## 四、问题与解决方案

### 4.1 集成测试重复运行失败

**问题：** 第二次运行集成测试时，步骤 5「患者授权医生」报错 `code=1003：已向该医生发起过授权，不可重复添加`。

**原因：** 测试脚本未清理上次残留的授权记录。

**解决方案：** 在 `backend/test/integration.test.js` 中新增 `requestRaw()` 函数，在授权前查询并撤销旧授权记录，再发起新授权。修复后测试可稳定重复运行。

### 4.2 本地后端启动端口冲突

**问题：** 运行 `node app.js` 时报 `EADDRINUSE: address already in use 0.0.0.0:3000`。

**原因：** 已有其他 Node 进程占用 3000 端口。

**解决方案：** 使用 `netstat -ano | findstr :3000` 查找 LISTENING 状态的 PID，然后 `taskkill /PID <PID> /F` 结束进程，再重新启动。
