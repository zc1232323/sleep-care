## 截图③a：Postman — 日视图 (period=day)

### 详细操作步骤

1. 打开 Postman
2. 先发一个登录请求拿 token：
   - `POST http://xxx:3000/api/user/login`
   - Body raw JSON: `{ "phone": "13800138000", "password": "123456" }`
   - 复制返回的 `data.token`
3. 新建请求：
   - Method: `GET`
   - URL: `http://你的IP:3000/api/sleep/summary?period=day`
   - Headers → Key: `Authorization`，Value: `Bearer <粘贴token>`
4. 点击 **Send**
5. 截图 **Postman 完整窗口**（URL栏 + Headers + 响应Body）

### 期望响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "period": "day",
    "labels": ["06/23", "06/24", "06/25", "06/26", "06/27", "06/28", "06/29"],
    "scores": [82, 75, 88, 70, 91, 84, 77],
    "avg_score": 81.0,
    "count": 7
  }
}
```

**验证要点**：
- code = 0（成功）
- labels 数组长度 = 7（近7天）
- scores 数组长度 = 7
- avg_score 在 60-95 之间
- period = "day"

## 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| Postman 返回 code=1001 | 未登录/无设备 | 先调用 login 接口，确保 devices 表有数据 |
| 小程序趋势图空白 | summaryData 为 null | 检查后端是否运行，baseUrl 是否正确 |
| markLine 位置不对 | avgPct 计算偏差 | 检查后端返回的 avg_score 是否在 40-100 范围 |
| 切换视图数据不变 | switchPeriod 未触发 | 检查 WXML 按钮 bindtap 是否正确绑定 |
| 评分全是同色 | 模拟数据波动小 | 正常，模拟数据 60-95 范围会有多种颜色 |
| 小程序颜色错乱 | CSS 未同步 | 确保 miniprogram/pages/report/report.wxss 是最新的 |
