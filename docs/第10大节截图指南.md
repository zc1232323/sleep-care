## 截图③a：Postman — PUT /api/doctor/confirm（确认授权）

### 操作

1. 用医生账号登录（13811111111/123456）获取 token：
   - POST `http://127.0.0.1:3000/api/v1/auth/login`
   - Body: `{"phone":"13811111111","password":"123456"}`
   - 复制 token

2. 新建请求：
   - Method: **PUT**
   - URL: `http://127.0.0.1:3000/api/doctor/confirm`
   - Headers:
     - `Authorization: Bearer <医生token>`
     - `Content-Type: application/json`
   - Body: 需要 patient_id 或 auth_id

**如何获取 auth_id？**

先执行截图③b的 GET /api/doctor/patients，从返回数组中找到 `id` 字段（授权记录ID）。

Body:
```json
{
  "auth_id": 1
}
```

3. 点 Send

### 期望响应

```json
{
  "code": 0,
  "message": "确认授权成功",
  "data": {
    "auth_id": 1,
    "patient_id": 2,
    "status": "active"
  }
}
```

**验证要点**：status = "active" ✅

## 截图③b：Postman — GET /api/doctor/patients（患者列表）

### 操作

1. Method: **GET**
2. URL: `http://127.0.0.1:3000/api/doctor/patients`
3. Headers: `Authorization: Bearer <医生token>`（不要 Body，不要 Content-Type）
4. 点 Send

### 期望响应

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 1,
      "patient_id": 2,
      "patient_name": "用户8000",
      "patient_phone": "13800138000",
      "status": "active",
      "latest_score": 75,
      "expire_date": "2026-07-07",
      "doctor_note": ""
    }
  ]
}
```

**验证要点**：返回数组，含 `status` 和 `latest_score` ✅

## 截图③c：Postman — GET /api/doctor/patient/data（active 可查看）

### 操作

1. Method: **GET**
2. URL: `http://127.0.0.1:3000/api/doctor/patient/data?patient_id=2`
   - 把 `2` 换成你的患者 patient_id
3. Headers: `Authorization: Bearer <医生token>`
4. 点 Send

### 期望响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "sleep_score": 75,
    "total_sleep_minutes": 420,
    "deep_sleep_minutes": 110,
    "light_sleep_minutes": 220,
    "rem_sleep_minutes": 90,
    "awake_minutes": 15,
    "awake_count": 2,
    "deep_ratio": 26,
    "sleep_stages": [...],
    "noise": {...}
  }
}
```

**验证要点**：active 患者返回完整报告数据 ✅

## 截图③d：Postman — GET /api/doctor/patient/data（pending 无权查看）

### 操作

1. 用患者账号再添加一个新医生授权（先不要确认）
2. 用新医生的 token 请求这个患者的 data
3. 或者：如果数据库里还有 pending 的授权记录，直接用那个医生的 token 测试

期望响应：
```json
{
  "code": 1003,
  "message": "该患者尚未授权完成，无权查看"
}
```

**验证要点**：pending 患者返回"无权查看" ✅

## 常见问题

| 问题 | 解决 |
|------|------|
| 访问 `http://localhost:3000/doctor.html` 404 | 检查 app.js 是否有 `app.use(express.static('public'))` 且后端已重启 |
| 登录返回"用户不存在" | 医生账号未创建，运行 `node create-test-doctor.js` 后重启后端 |
| 确认授权返回"未找到待确认记录" | 患者还没发起授权，或该授权已经被确认/撤销 |
| 查看报告无数据 | 患者需要先生成睡眠报告（调用一次 GET /api/sleep/report/daily） |
| Content-Type 错误 | 必须是 `application/json`（不要写成 `application`） |
