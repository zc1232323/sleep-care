### 截图 3-③：PUT /api/doctor/note 保存建议成功响应

#### 前置

需要一条 **active** 的授权记录。如果没有，按以下步骤创建：

1. 用患者手机号登录（Postman 用 `登录` 请求，改手机号）。
2. 用患者 token 调 `POST /api/doctor/grant`，Body 为 `{ "doctor_id": 9 }`（9 为测试医生 id）。
3. 用医生 token 调 `PUT /api/doctor/confirm`，Body 为 `{ "auth_id": <上一步返回的id> }`。

完成后，用 `GET /api/doctor/patients` 查看 `patient_id` 字段（即保存建议时用的患者 ID）。

#### 操作

1. 选择 **医生端 → 保存干预建议**。
2. 确认请求信息：
   - Method: `PUT`
   - URL: `{{baseUrl}}/api/doctor/note`
   - Headers:
     - `Authorization: Bearer {{doctorToken}}`
     - `Content-Type: application/json`
   - Body (raw JSON):

```json
{
  "patient_id": 10,
  "note": "建议保持规律作息，睡前避免使用手机。"
}
```

> 把 `10` 换成你实际数据库里的 `patient_id`。

3. 点击 **Send**。

#### 期望响应

```json
{
  "code": 0,
  "message": "建议保存成功",
  "data": {
    "patient_id": 10,
    "note": "建议保持规律作息，睡前避免使用手机。"
  }
}
```

#### 截图要点

- Body 中的 `patient_id` 和 `note` 内容。
- Response 中 `code: 0`、返回的 `note` 与请求一致。

## 九、常见翻车点与处理

| 现象 | 原因 | 处理 |
|---|---|---|
| 注册医生返回“该手机号已注册” | 数据库已有该号码 | 换末尾几位数字重试 |
| 医生列表返回空数组 | 没有 role='doctor' 的用户 | 先执行“注册医生” |
| 保存建议返回“未找到有效的授权记录” | 授权状态不是 active 或 patient_id 错误 | 先确认授权，再用正确的 patient_id |
| 获取建议返回空字符串 | 该 patient_id 没有保存过建议 | 先执行保存建议 |
| 小程序看不到医生列表 | 后端没启动或 baseUrl 错误 | 检查后端运行，检查 app.js 中 `baseUrl` |
| 医生端 Web 404 | 静态文件服务未启用 | 确保 backend/app.js 有 `app.use(express.static('public'))` 且已重启 |
| SQLite 查看中文乱码 | 终端编码问题 | 用 VSCode SQLite 插件查看，不要在命令行 cat |
| Postman 中昵称显示乱码 | 之前用 cmd/curl 注册（GBK 编码）导致存量数据异常 | 用 **新手机号** 在 Postman 里重新注册，Postman 用 UTF-8 就不会乱码 |
