## 二、缺失内容详细分析

### 🔴 严重缺失（影响讲义验收）

#### 1. 缺少2张数据库表（`backend/db/schemas.js`）

讲义第1大节要求5张核心表，当前项目只有3张：

| 表名 | 用途 | 讲义要求 | 当前状态 |
|------|------|---------|---------|
| users | 用户信息 | ✅ 已创建 | ⚠️ 字段不全 |
| devices | 设备信息 | ✅ 已创建 | ⚠️ 字段不全 |
| sleep_reports | 睡眠报告 | ✅ 已创建 | ⚠️ 字段不全 |
| **user_settings** | **用户设置（作息等）** | **❌ 缺失** | **未创建** |
| **doctor_authorizations** | **医生授权** | **❌ 缺失** | **未创建** |

需要在 `schemas.js` 中补充以下建表语句：

```sql
-- user_settings 表（一对一）
CREATE TABLE IF NOT EXISTS user_settings (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL UNIQUE,
  bed_time TIME,
  wake_time TIME,
  sunrise_duration_minutes INTEGER DEFAULT 10,
  sound_type VARCHAR(20),
  brightness_level INTEGER DEFAULT 5,
  volume_level INTEGER DEFAULT 5,
  timezone VARCHAR(50) DEFAULT 'Asia/Shanghai',
  dnd_enabled INTEGER DEFAULT 0,
  dnd_start TIME,
  dnd_end TIME,
  created_at TEXT DEFAULT (datetime('now')),
  updated_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- doctor_authorizations 表
CREATE TABLE IF NOT EXISTS doctor_authorizations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  patient_id INTEGER NOT NULL,
  doctor_id INTEGER NOT NULL,
  status VARCHAR(20) DEFAULT 'pending',
  expire_date DATE,
  doctor_note TEXT,
  requested_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  responded_at DATETIME,
  created_at TEXT DEFAULT (datetime('now')),
  updated_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (patient_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (doctor_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### 🟡 中等差异（功能可用但不完全对齐讲义）

#### 5. API 路径不一致

| 讲义要求路径 | 项目实际路径 | 差异 |
|-------------|-------------|------|
| POST /api/auth/register | POST /api/v1/auth/register | 多了 v1 |
| POST /api/auth/login | POST /api/v1/auth/login | 多了 v1 |
| GET /api/device/list | GET /api/v1/devices/list | 多了 v1，device→devices |
| POST /api/device/add | POST /api/v1/devices/add | 多了 v1，device→devices |
| PUT /api/devices/:id | PUT /api/v1/devices/:id | 多了 v1 |
| DELETE /api/devices/:id | DELETE /api/v1/devices/:id | 多了 v1 |
| GET /api/sleep/report/daily | GET /api/sleep/report/daily | ✅ 一致 |

> 注：小程序前端已适配了 v1 路径，前后端是通的。但如果老师按讲义路径验收，可能不通过。

#### 6. 缺少 `dbGetOne` / `dbGetAll` 辅助函数

讲义要求在 app.js 中定义 `dbGetOne` 和 `dbGetAll` 辅助函数，当前项目使用 `db.exec()` 直接查询，功能等效但实现方式不同。

#### 7. 缺少 `setToken()` / `getToken()` 方法

讲义要求 `miniprogram/app.js` 中定义 `setToken()` 和 `getToken()` 方法，当前项目直接使用 `wx.setStorageSync` / `wx.getStorageSync`，功能等效但封装不同。

#### 8. 设备添加接口缺少 `is_virtual` 参数

讲义要求 `POST /api/device/add` 支持 `is_virtual` 参数（布尔值）和手动输入 `device_serial`，当前项目只支持自动生成虚拟设备。

#### 9. 睡眠报告未关联设备

讲义要求获取用户第一台设备的 `device_id`，当前项目的 `reports.js` 中 `device_id` 始终为 NULL。
