## 三、常见问题：测试手机号已注册怎么办？

如果运行测试时报错 `该手机号已注册`，说明之前已跑过测试。按以下步骤重置：

1. **关闭后端**：在终端按 `Ctrl + C`，或任务管理器结束 node 进程。
2. **删除测试用户**：用 Python 清理数据库（复制到 VSCode 终端执行）：

```bash
C:\Users\z\.workbuddy\binaries\python\versions\3.13.12\python.exe -c "
import sqlite3
db = sqlite3.connect(r'C:\Users\z\Desktop\sleep-care\sleep_care.db')
for phone in ['18800000001', '18800000002']:
    db.execute('DELETE FROM users WHERE phone = ?', (phone,))
    print('deleted', phone)
db.commit(); db.close()
"
```

3. **重新启动后端**：

```bash
cd backend
node app.js
```

4. **重新运行测试**：

```bash
node backend/test/integration.test.js
```

> 小提示：`backend/test/integration.test.js` 已做了兼容处理：如果手机号已注册会自动改为登录，但**首次运行截图最干净**，建议重置后截图。

## 五、小程序 setData 优化说明（不单独截图，但要知道在哪）

作业自检清单第 4 条要求「小程序 home.js 等页面已合并多次 setData」。优化后的位置如下，供你检查：

- `pages/home/home.js`：
  - `onPullDownRefresh()` 合并 `loading + refreshing` 为一次 `setData`
  - `loadDailyReport(skipLoading)` 避免重复设置 `loading`
- `pages/report/report.js`：
  - `handleStagesRes()` 合并 `stagesData`、`xLabels`、`stageSource`、`summary`、`loading` 为一次 `setData`
  - `handleNoiseRes()` 合并 `noiseData`、`noiseXLabels`、`noiseSource`、`loading` 为一次 `setData`
  - `loadSummary()` 合并 `summaryData`、`loading` 为一次 `setData`
- `pages/doctors/doctors.js`：
  - `loadGrantedDoctors()` 在 `success` 中同时设置 `grantedDoctors` 和 `loading: false`
  - `addDoctor()` 移除 `success` 与 `complete` 中重复的 `adding: false`

> 这些代码截图可以归入「截图 4」或单独截图提交，但通常老师只要求看到代码存在即可，建议把关键合并后的 `setData` 区域截 1~2 张作为补充。
