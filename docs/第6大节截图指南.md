## 截图③：Postman 噪音接口测试（3张）

### ③-① 第一次请求噪音接口

1. 打开 **Postman**，新建 GET 请求
2. URL：``http://localhost:3000/api/sleep/report/noise?date=2026-06-30``
3. **Authorization** 标签页 → Type: `Bearer Token` → 粘贴你的 JWT Token
4. 点 **Send**

**验证响应 JSON：**
- `"count": 144` ✓
- `"source": "generated"` ✓（第一次）
- noises 数组中检查数值范围：
  - 夜间时间点（22:00~06:00）：`value` 应在 **30~40** dB 之间
  - 白天时间点（06:00~22:00）：`value` 应在 **45~65** dB 之间

### ② 第二次请求同一日期（不重启后端）

1. 保持 URL 和 Token 不变
2. 再点一次 **Send**

**验证：**
- `"source": "db"` ✓（从数据库读取）
- noises 数组的值与第一次完全一致 ✓

### ③ 用 SQLite 查看 noise_json 字段

**方式A（推荐）：双击运行 `backend/查看数据库.bat`**
- 双击 `backend\查看数据库.bat`
- 输入表名：`sleep_reports` 回车
- 找到最新日期的记录，看 `noise_json` 字段是否已存入 144 个数据点
- 应该能看到类似：`[{"time":"20:00","value":56.8","period":"day"},{...`

**方式B：命令行（需要用完整 Python 路径）**
```bash
C:\Users\z\.workbuddy\binaries\python\versions\3.13.12\python.exe C:\Users\z\Desktop\sleep-care\backend\view_db.py sleep_reports
```
> ⚠️ **注意**：Windows CMD 中不能直接写 `python`，必须用完整路径！
