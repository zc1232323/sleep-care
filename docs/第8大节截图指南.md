## 截图③a：Postman — GET 首次获取设置（默认值）

### 操作

1. 先确保登录拿 token（如果 token 过期了重新登录）
2. 新建请求：
   - Method: **GET**
   - URL: `http://127.0.0.1:3000/api/setting/plan`
   - Headers → `Authorization: Bearer <你的token>`
3. 点 Send

### 期望响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "bed_time": "23:00",
    "wake_time": "07:00",
    "sunrise_duration_minutes": 10
  }
}
```

**验证要点**：
- code = 0 ✅
- 返回默认值 23:00 / 07:00 / 10 ✅

---

## 截图③b：Postman — PUT 保存新设置

### 操作

1. 改 Method 为 **PUT**
2. URL 不变：`http://127.0.0.1:3000/api/setting/plan`
3. Headers 加多一条：`Content-Type: application/json`
4. Body → raw → JSON，填入：

```json
{
  "bed_time": "22:30",
  "wake_time": "06:30",
  "sunrise_duration_minutes": 20
}
```

5. 点 Send

### 期望响应

```json
{
  "code": 0,
  "message": "设置保存成功",
  "data": {
    "bed_time": "22:30",
    "wake_time": "06:30",
    "sunrise_duration_minutes": 20
  }
}
```

## 截图③c：Postman — GET 验证数据已更新

## 常见问题

| 问题 | 解决 |
|------|------|
| Postman GET 返回 code=1001 | 重启后端（新增路由需要重启才生效） |
| Postman 返回 401 | Token 过期，重新登录拿新 token |
| 设置页 slider 拉不动 | 检查 bindchange="onSunriseChange" 绑定 |
| 保存后重新进入变回默认值 | 后端 PUT 没生效，检查 console 有无报错 |
| 首页没有"作息设置"按钮 | 重新编译小程序 |
