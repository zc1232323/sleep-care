# 第13大节 部署成功完整步骤（实际操作记录）

> 本文档记录了 SleepCare 项目从零到部署成功的每一步实际操作，供视频录制和复习参考。

---

## 第一步：创建 CloudBase 环境

1. 访问 [CloudBase 控制台](https://console.cloud.tencent.com/tcb)
2. 点击「新建环境」
3. 环境名称：`sleep-care`
4. 套餐选择：**免费体验版**
5. 创建成功后，记录环境 ID：`sleep-care-d7gd3tdy9fdaa87e9`

---

## 第二步：开通 MySQL 数据库

1. 在 CloudBase 控制台 → 数据库 → MySQL 数据库 → 点击「开通」
2. 选择 **免费版**
3. 设置数据库用户名：`sleepcare`，密码：`SleepCare@2026`
4. 开通成功后，数据库默认库名为环境 ID：`sleep-care-d7gd3tdy9fdaa87e9`
5. 记录连接信息：
   - 内网地址：`172.17.0.2:3306`
   - 外网地址：`sh-cynosdbmysql-grp-0kkqdf2y.sql.tencentcdb.com:23088`（需手动开启）

---

## 第三步：创建数据库表（5 张）

1. 进入 CloudBase 控制台 → MySQL 数据库 → **SQL 编辑器**
2. 逐条执行 `docs/migration-cloudbase.sql` 中的建表语句
3. 注意：**CloudBase SQL 编辑器一次只能执行一条语句**
4. 执行完后验证：

```sql
SHOW TABLES;
```

预期看到 5 张表：`users`、`devices`、`sleep_reports`、`user_settings`、`doctor_authorizations`

---

## 第四步：部署后端到云托管

1. 进入 CloudBase 控制台 → 云托管
2. 点击「新建服务」
3. 服务名称：`sleepcare-api`
4. 部署方式：**GitHub 仓库**
5. 绑定 GitHub 仓库，选择 `main` 分支
6. 端口：`3000`
7. 开启 **「内网默认地址」**（网络配置）
8. 点击「创建并部署」，等待 3-5 分钟

---

## 第五步：配置环境变量

在云托管服务详情页 → 服务配置 → 环境变量，配置以下 JSON：

```json
{
  "DATABASE_TYPE": "mysql",
  "DB_HOST": "sh-cynosdbmysql-grp-0kkqdf2y.sql.tencentcdb.com",
  "DB_PORT": "23088",
  "DB_USER": "sleepcare",
  "DB_PASSWORD": "SleepCare@2026",
  "DB_NAME": "sleep-care-d7gd3tdy9fdaa87e9",
  "JWT_SECRET": "sleep-care-secret-key-2026",
  "PORT": "3000"
}
```

> **关键**：内网地址 `172.17.0.2` 在 CloudBase Run 中无法访问，必须开启 MySQL **外网地址**，使用 `sh-cynosdbmysql-grp-0kkqdf2y.sql.tencentcdb.com:23088`。

---

## 第六步：设置最小实例数

1. 服务详情 → 服务配置 → 副本配置
2. **最小副本数**从 `0` 改成 `1`
3. 最大副本数保持 `5`
4. 点击「部署」

> 避免冷启动导致 503 Service Unavailable。

---

## 第七步：创建 HTTP 网关路由

1. CloudBase 控制台 → HTTP 网关
2. 点击「新建路由」
3. 路径填：`/`
4. 目标服务选择：`sleepcare-api`
5. 保存后获得公网域名：`https://sleep-care-d7gd3tdy9fdaa87e9-1449296628.ap-shanghai.app.tcloudbase.com`

> 不创建 HTTP 网关，直接访问默认域名会报 `INVALID_HOST`。

---

## 第八步：推送代码到 GitHub

**每次修改代码后，用 VSCode 推送**（不是 Git Bash，Git Bash 会报凭证错误）：

1. 打开 VSCode → 左侧 Git 面板
2. 输入提交信息
3. 点击「提交」
4. 点击「同步/push」
5. CloudBase 会自动检测到新代码并重新部署

## 踩过的坑（总结）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| MySQL 连接超时 ETIMEDOUT | 内网地址 172.17.0.2 不可达 | 开启 MySQL 外网地址，使用域名连接 |
| 注册返回「注册失败」 | db.exec/db.run 缺少 await | 所有数据库调用前加 await |
| 注册返回「已注册」 | MySQL wrapper 空结果返回非空数组 | 空结果时返回 [] |
| 医生授权失败 | datetime('now','localtime') 是 SQLite 语法 | 改为 JS 生成 ISO 时间字符串 |
| 医生端中文乱码 | express.static 未指定 charset | setHeaders 强制 charset=utf-8 |
| 503 Service Unavailable | 最小实例数为 0，冷启动超时 | 最小实例数改为 1 |
| INVALID_HOST | 默认域名需 HTTP 网关 | 创建 HTTP 网关路由 |
| Git Bash push 失败 | 凭证问题 | 用 VSCode 终端推送 |
